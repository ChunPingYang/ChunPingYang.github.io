    Distribution System – Lamport’s Mutual Exclusion Algorithm 01/2020-05/2020

    • Motivation

      • Nowadays, Google document and Google Drive are very popular everywhere. People can be access the same files and edit them together simultaneously. The development of the distribution system leads to the popularity of the application. I would like to know more about how Lamport’s Mutual Exclusion Algorithm and multithreaded programming are applied in the industry.

    • Requirement

      • Design and implement a distributed system which consists ofthreeserver processes andfiveclient processes. Assume that each file is replicated on all the servers, and all replicas of a file are consistent in the beginning. A client can perform a READ or WRITE operation on the files. READ operation involves only one server that has the target file and the server is chosen randomly by a client. WRITE involves all servers that have a copy of the target file and all of them should be updated in order to keep the replicas consistent. READ/WRITE on a file can be performed by only one client at a time. However, different clients are allowed to concurrently perform a READ/WRITE on different files. In order to ensure this, implement Lamport’s mutual exclusion algorithm among clients so that no READ/WRITE violation could occur. The operations on files (hosted by servers) can be seen ascritical sectionexecutions. To host files, create separate directory for each server. The servers must support following operations and reply to the clients with appropriate messages –

      1. ENQUIRY : A request from a client for information about the list of hosted files.
      2. READ: A request to read last line from a given file
      3. WRITE: A request to append a string to a given file

    • Tech Stack

      • Java, Multithreaded Programming, Socket Programming

    Search Engine 08/2019-01/2020

    • Motivation

      • Previously, I used Alibaba commercial platform to help my home company to sell products. I set up lots of relevant industry key words to try to make our products to show up in the front page As close as possible. The more relevant industry key words, the more possible customers can find your products. I am very curious how the search engine works. Based on the things I learned on software architecture course, we tried to build a simple search engine.

    • Technique

      • Search engine is composed of three parts. Crawler, Indexing, and Search.

      We use crawler to crawl every page its content to files. Each page contains its own content. And each page corresponds to a file. And then we can get the content as a string, assign the first word as first index, calculate difference between each word and the first word, and then put each word as the first word, and move every word before the first word to the last. You will get multiple different permutation sentences. This is CircularShift Index. When you do this task, you also need to remove noise words in the meanwhile.

      After finishing this task, let’s get start to do AlphabetShift index. We tried to sort each sentence by alphabetical order, to become a dictionary. The previous indexes we calculate all corresponds to each sentence. Then we can use this result to do searching.

    • Weakness

      • Space complexity is not so good. When you do CircularShift, you need to copy all the data. Similarly, when you do AlphabetShift, you also need to copy all the data of CircularShift. When you copy the data, you have to reconstruct data structure to store those data, it would take some time.
